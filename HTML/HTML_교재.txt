<1> HTML 이 뭡니까?
■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■
●[웹브라우저]라는 SW에서 [문자열], [이미지], [동영상], [입력양식] 등을 출력하기 위해 사용는 언어를 말한다.
●HyperText Markup Language 의 약어이다.
●HTML 은 실무적으로 [웹브라우저]가 [웹서버]에 접속한 후 [웹서버]가 [웹브라우저] 쪽으로 던져주는 언어입니다.
  HTML 안에는 DB 연동 결과물이 내재되어 있다.
●HTML 은 프로그램 언어는 아니다. 로직 같은 형식이 없습니다.
●<주의>개발자는 [입력양식] 태그를 무지 많이 조심해야한다. 오라클과 연결되고 Spring 하고도 연결되어 있다.

<2> HTML 차원에서 태그(tag)가 뭡니까?
■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■
●HTML 안에서 문자열, 이미지, 동영상, 입력양식 등을 출력하기 위해 사용되는 형식을 말한다.
----------------------------------
●태그(tag) 의 형식1
----------------------------------
  ----------------------------------
  <태그명 속성명="속성값">
    컨텐츠
  </태그명>
  ----------------------------------
  ▶<태그명> 과 </태그명> 사이의 컨텐츠가 태그 기능의 영향을 받는다.
  <예> <font color="blue">Bigdata</font>   => 문자열 Bigdata 가 파랑색으로 웹브라우저에 출력됨.
  <예> <b>Bigdata</b>                      => 문자열 Bigdata 가 두꺼워져 웹브라우저에 출력됨.
  <예> <center>Bigdata</center>            => 문자열 Bigdata 가 웹프라우저의 중앙위치로 출력됨.
----------------------------------
●태그(tag) 의 형식2
----------------------------------
  <태그명 속성명="속성값"/>
  ----------------------------------
  <예> <hr/>      => 수평선 출력됨.
  <예> <input type="text" name="uid" size="10"> => 한줄의 문자열을 입력하는 입력양식 출력
----------------------------------
  ▶태그명 --> 태그 종류명을 말한다.
	▶속성명 --> 태그의 성질을 말한다.
	▶속성값 --> 속성명에 대한 값을 말한다.
	▶속성명="속성값" 은 0개 이상 나올 수 있다.
	▶속성값은 " 또는 ' 로 감싼다. 만약 속성값 중간에 공백이 없으면 " 또는 '는 생략할 수 있으나 되도록 사용한다.
	▶컨텐츠에는 문자열 또는 또 다른 태그가 나올 수 있다.
	▶태그명, 속성명은 대문자 또는 소문자를 써도 된다. 되도록 소문자를 사용한다.

<4> HTML의 기본구조 관련 태그는?
■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■
------------------
<html>...</html>
------------------
  ▶HTML 소스 전체를 감싸는 태그.
------------------
<head>...</head>
------------------
  ▶HTML 문서의 제목이나 머리말을 감싸는 태그. 자바스크립트(Javascript) 함수가 삽입되는 곳
------------------
<title>...</title>
------------------
  ▶웹 브라우저의 제목 표시줄에 나타나는 내용을 감싸는 태그.
  ▶<head> 태그 안에 삽입된다.

------------------
<body>...</body>
------------------
  ▶웹브라우저 화면에 출력될 컨텐츠(문자열, 이미지, 입력양식 등)를 감싸는 태그

------------------
<meta>
------------------
  ▶[화면이동], [문자셋], [검색 엔진이 검색하는 문자열] 등을 설정하는 태그이다.

<6> HTML 에서 사용하는 색상값 표현 방법은 뭐죠?
■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■
  -----------------------------------------------------
  <방법1> 영문표기법
  -----------------------------------------------------
    ▶영문으로 색상을 표기하는 방법
    ▶<예>black, blue, yellow, white, red, gray, green, purple, aqua, olive, silver, navy,
          fuchsia, lime, maroon, teal 등
  -----------------------------------------------------
  <방법2>RGB(Red Green Blue) 표기법
  -----------------------------------------------------
    ▶Red 표현 16진수 2자리, Green 표현 16진수 2자리, Blue 표현 16진수 2자리를
      붙여 색상을 표기하는 방법
    ▶총 16,777,216 개의 색상을 지원
    ▶<예>#000000(black), #FFFFFF(white), #FF0000(red), #00FF00(green), #0000FF(blue)
    ▶실무적으로는 영문표기법 보다 RGB 표기법을 더 많이 쓴다.

<7> HTML 에서 사용하는 길이 단위에는 뭐가 있죠?
■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■
  %, px, pt 정도만 기억하면 된다.
  -----------------------------------------------------
  절대단위
  -----------------------------------------------------
    in, cm, mm,
    pt => 포인트. 1포인트=1/72인치.
    pc => 파이카. 1파이카=12포인트
  -----------------------------------------------------
  상대단위
  -----------------------------------------------------
    %   => 웹화면의 좌우 또는 상하를 100%로 놓고 설정하는 단위.
    px  => 스크린의 1픽셀을 1로 함.
    em  => 웹브라우저의 기본 폰트 크기(16px)을 1로 하는 단위.
           소수점 표현도 가능. 예>0.85em

  즉, 파일호출자 입장에서 피호출파일을 호출할 때 피호출파일의 경로를 어떻게 설정해야하나?

  C:a/b/YYY.html     => "YYY.html"

  C:a/zzz.html       => "../zzz.html"

<8> 웹서버 구축 후 웹서버에 내장된 파일끼리, 즉, 웹프로그램 자원끼리 호출할 때 자원경로 종류?
■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■
  *******************************************************************
  절대경로 예
  *******************************************************************
    -------------------------------------------------------------------
    "http://~:8080/a/b/xxx.jsp"    => 전세계 유일한 URL 주소인 "http://~" 에 존재하는
                                      웹서버 내에 a/b/xxx.jsp 경로에 있는 xxx.jsp 파일 호출하기
                                      포트번호 앞까지의 주소를 도메인 주소라고 부른다.
                                      포트번호 뒤는 웹서버 내의 주소를 말한다.
                                      포트번호 대개 생략된다.
                                      도메인 주소는 대부분 com 또는 net 또는 co.kr 또는 go.kr 등으로 끝난다.
    -------------------------------------------------------------------
    "/a/b/xxx.jsp"    => 동일한 웹서버 내에서 최상위 폴더 밑에 /a/b/xxx.jsp 경로에 있는 파일 호출하기
    -------------------------------------------------------------------
  *******************************************************************
  상대경로 예
  *******************************************************************
    -------------------------------------------------------------------
    "a/b/xxx.jsp"     => 동일한 웹서버 내에서 호출 파일이 속한 폴더 내에 a 폴더내에 b폴더 내에 피호출 파일 xxx.jsp 파일 호출하기
    -------------------------------------------------------------------
    "../xxx.jsp"      => 동일한 웹서버 내에서 호출 파일이 속한 폴더를 벗어나서 존재하는 피호출 파일 xxx.jsp 파일 호출하기
    -------------------------------------------------------------------
    "../d/xxx.jsp"    => 동일한 웹서버 내에서 호출 파일이 속한 폴더를 벗어나서 존재하는 d 폴더내에 피호출 파일 xxx.jsp 파일 호출하기.
    -------------------------------------------------------------------
    "xxx.jsp"    => 동일한 웹서버 내에서 호출 파일이 속한 폴더 안에 존재하는 피호출 파일 xxx.jsp 파일 호출하기.
    -------------------------------------------------------------------

<9> 웹 개발 시 [웹디자이너], [웹 퍼블리셔], [웹프로그래머]의 역할은 뭐죠?
■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■
●디자이너
  ▶포토샵, 일러스트레이터를 사용하여 화면에 출력되는 이미지의 그래픽 작업을 한다.
●웹 퍼블리셔
  ▶HTML, CSS, Javascript를 사용하여 웹하면의 UI(=User Interface) 제작 작업을 한다.
  ▶<참고>Javascript는 프로그래밍 언어의 한 종류이므로 [웹 퍼블리셔]는 프로그래머로서의 역량이 더 많이 필요하다고 볼 수도 있다.
●웹프로그래머
  ▶서버 쪽 언어(JSP, Java, PHP, ASP, .NET)을 사용하여 UI로부터 또는 UI로의 데이터의 이동, DB 연동 작업을 한다.
  ▶입력양식 관련 HTML 은 개발자가 할 수 도 있다.
  ▶복잡하고 어려운 Javascript 작업을 하는 경우도 있다.

<11> 글자 관련 태그 관련 태그에는 뭐가 있죠?
■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■
-------------------
<h~>문자열</h>
-------------------
  ▶~ 자리에 1~6 까지의 숫자를 넣어 문자열의 크기를 조절한다. (1 -> 6 수록 작다)
  ▶문자열이 한행을 독차지하고 전후에 행도 비운다. 즉 3행을 독차지 한다.
  ▶자동으로 문자열이 두꺼워진다.

    <예시>
    <h1>에버랜드</h1>
    <h2>에버랜드</h2>
    <h3>에버랜드</h3>
    <h4>에버랜드</h4>
    <h5>에버랜드</h5>
    <h6>에버랜드</h6>

-------------------
<font>문자열</font>
-------------------
  ---------------------------------------------------------
  ▶문자열의 글꼴, 크기, 색상 등을 조절한다.
  ---------------------------------------------------------
  ▶<font size="글자크기" face="글꼴명" color="글색상">에버랜드</font>
  ---------------------------------------------------------
    => 글자크기 1~7 만 사용. 숫자 클수록 글자가 크다.
    =><주의>생략 시 size="3" 설정.
-------------------

<em>문자열</em>
<cite>문자열</cite>
<var>문자열</var>
-------------------

-------------------
<code>문자열</code>
<kbd>문자열</kbd>
<tt>문자열</tt>
-------------------
	▶문자열을 타자기체로 바꾼다.
-------------------
<b>문자열</b>

-------------------
	▶문자열을 굵은 글씨로 바꾼다.

	▶문자열 가운데 선을 긋는다.

<14> div 태그가 뭐 할 때 사용하나요?
■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■
●이미지, 문자열 등을 기준별로 그룹화 시켜 서로 [겹치게 포개 놓기], [펼쳐놓기], [감춰 놓기]
●<div>~</div> 영역은 block 형식(=한 행을 차지함=아래로 쌓임)이다. inline 형식 =>  오른쪽으로 쌓임
●<div> 태그의 속성은 CSS의 표현방식인 style 속성으로 표현한다.
  ----------------------------------
  ▶<div styl="속성1:속성값1;~;">
  ----------------------------------
●style 속성
  position:static;    => div 태그의 현재 코딩 위치서 그대로 출력됨. 디폴트 설정이다. 안써도 됨.
  position:relative;  => div 태그의 현재 코딩 위치를 0,0 으로 기준 삼고 x축 y축으로 이동하여 다른 컨텐츠 위에 위치한다.
  position:absolute;  => 화면 왼쪽상단 모서리를 0,0 으로 기준 삼고 x축 y축으로 이동하여 다른 컨텐츠 위에 위치한다.
  left:화면왼쪽여백;  => 웹브라우저 왼쪽 테두리에서 div 태그 영역(div 태그)까지의 가로 위치를 나타냄.
                         이게 먹힐려면 position:absolute; 또는 position:relative; 가 나와야한다.
  top:화면위쪽여백;   => 웹브라우저 위쪽 테두리에서 div 태그 영역까지의 세로 위치를 나타냄.
                         이게 먹힐려면 position:absolute; 또는 position:relative; 가 나와야한다.

  padding:상여백 우여백 하여백 좌여백;
                      => div 태그영역 내부의 상,우,하,좌, 여백 지정
                      => 상,우,하,좌 여백이 같은 값이면 padding:여백; 로만 써도 된다.
                      => 상,우,하,좌 여백을 따로 주고싶으면 아래처럼 하면된다.
                        padding-top:여백;
                        padding-right:여백;
                        padding-bottom:여백;
                        padding-left:여백;

  width:div태그좌우영역넓이;
  height:div태그위아래영역넓이;

  border:테두리선두께 선모양 색상;
    => div 태그 영역의 테두리선두께, 선모양, 색상 설정. 예> border:2px solid red

  background-color:div태그영역색상;

  z-index:우선순위번호;
    => div 태그 영역이 겹쳤을 때 우선순위를 지정. 번호가 클수록 상위.

  visivility:보임여부설정;
    => div 태그 영역 보임여부를 설정. 보임여부 설정값은 visible(보임), hidden(안보임)
       <주의>hidden 설정 시 안보이고 차지하는 공간은 남아 있음.

  display:보임여부설정;
    => div 태그 영역 보임여부를 설정. 보임여부 설정값은 block(보임), none(안보임)
       <주의>none 설정 시 안보이고 차지하는 공간도 없음.

<15> table 태그, td 태그, tr 태그가 뭐 할 때 사용하나요?
■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■
  문자열, 또는 이미지를 [행]과 [열]로 표시하는 태그다.
	즉 컨텐츠의 배치를 관리하는 태그이다.
	-----------------------------------------------------
	n행 m열의 행과 열을 만들어 컨텐츠를 배치하려면 아래 형식을 가진다.
	-----------------------------------------------------
	<table>
		<caption>표제목</caption>
		<thead>
			<tr><th>해더1</th><th>헤더2</th>~<th>헤더m</th></tr>
		</thead>
		<tbody>
			<tr><td>데이터11</td><td>데이터12</td>~<td>데이터1m</td></tr>
			<tr><td>데이터21</td><td>데이터22</td>~<td>데이터2m</td></tr>
			<tr><td>데이터31</td><td>데이터32</td>~<td>데이터3m</td></tr>
			.....
			<td>데이터n1</td><td>데이터n2</td>~<td>데이터nm</td>
		</tbody>
	</table>
	-----------------------------------------------------
	●<tr>의 개수가 전체 행의 개수이다.
	●한 행의 <td> 또는 <th>의 개수가 열의 개수이다.
  ●<caption> 태그는 생략이 가능하다.
  ●<thead>,<tbody> 는 생략 가능하다. 그러나jQuery 에서 테이블을 컨트롤 할 때는 유용하게 사용된다.
  ●</td></th></tr> 는 생략 가능하나 영역을 나타내므로 되도록 쓴다.

<16> [입력 양식] 관련 태그의 종류는? (입.)
■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■

●입력 양식 관련 태그는 [데이터]를 [입력] 또는 [선택]할 수 있는 태그 또는 그와 관련된 태그를 말한다.
●입력 양식 관련 태그에 입력 또는 선택한 데이터는 서버의 웹프로그래밍 언어로 전달되고 최종 DB=오라클(Oracle) 에 저장된다.
-----------------------------
●입력 양식 관련 태그의 종류
-----------------------------
  ---------------------------
  form 태그
  ---------------------------
    ▶여러 [입력양식]을 1개의 그룹으로 묶는 태그다. 마치 택배상자와 비슷한 역할을 한다.
    ▶서버의 URL 주소, 데이터 전송방법, 등을 설정할 수 있다.
    ▶실질적인 입력양식은 아니나, 중요한 태그이다. 
    -----------------
    ▶form 태그의 속성
    -----------------
      name   -> form 태그의 이름을 설정하는 속성명. form 태그가 여러개 있을 경우 구별하기위해 주는 속성이다.
      action -> form 태그 안의 입력양식에 입력(선택)된 데이터를 보낼 [서버 쪽 URL 주소]를 설정하는 속성이다.
      method -> 입력(선택) 데이터를 서버로 보내는 방법을 설정

                <예> method="get"  => 입력(선택) 데이터가 URL 뒤에 붙어 전송. 
                                      보안성이 낮은 데이터 전송 시 사용.
                                      이 방식으로 데이터를 전송할 때 URL 주소에 전송 데이터를 삽입만 하면 되므로 코딩이 줄어든다.
                                      즉, 입력양식이 필요가 없다.

                <예> method="post" => 입력(선택) 데이터가 안보이게 전송. 
                                      보안성이 높은 데이터 전송 시 사용.
                                      이 방식으로 데이터를 전송할 때 입력양식에 데이터를 삽입해야 되므로 코딩이 늘어난다.
                                      즉, 입력양식이 꼭 필요하다. 
  ---------------------------
  input
  ---------------------------
    ▶데이터를 입력 또는 선택(=체크)할 수 있는 [입력양식]을 출력하는 태그다.  
  ---------------------------
  select
  ---------------------------
    ▶1개 이상의 목록 중 하나를 선택하는 [목록상자]를 출력하는 태그다.  
    ▶목록을 감추고 클릭하면 목록을 볼 수 있어 공간을 적게 차지하는 장점이 있다.
  ---------------------------
  option
  ---------------------------
    ▶select 태그 내부에서 사용되어 [목록상자] 내부의 목록을 출력하는 태그다. 
  ---------------------------
  textarea
    ▶다중 행으로 문자열을 입력 할 수 있는 [입력 양식]을 출력하는 태그다.
